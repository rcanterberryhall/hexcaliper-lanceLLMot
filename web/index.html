Overview

LanceLLMot is a self-hosted AI workbench for safety engineering document analysis. It runs entirely on your local infrastructure via Ollama — no data leaves your network unless you explicitly approve it.

Upload ISO/IEC standards, contracts, THEOPs, FMEAs, hazard analyses, FAT/SAT test records, PLC code, and correspondence. The AI connects documents by shared safety concepts (diagnostic coverage, SIL, safe state, etc.) so you can ask cross-document questions like "Does this THEOP clause comply with IEC 61508 SIL 2?"

All analysis runs locally. Documents and conversations never leave your server unless you use the Cloud Escalation feature.

Scope hierarchy

Documents are organised in a three-tier hierarchy. When you chat with a project selected, all tiers above it are automatically included as context:

Global
ISO/IEC standards and shared reference material. Available in all conversations and all projects.
Client
Contracts, client requirements, and specifications that apply to all of that client's projects.
Project
THEOPs, FMEAs, hazard analyses, FAT/SAT records, and PLC code specific to one project. Inherits global + client documents.
Chat
Temporary documents uploaded during a single conversation. Deleted when the conversation is cleared.

Document classification

Public
Technical library documents (manufacturer manuals, standards). Eligible for auto-escalation when AUTO_ESCALATE=true.
Client
Any client or project file, or anything sourced from M-Files. Cloud escalation always requires explicit manual approval. Never exposed in the public library view.

Classification can be set at upload time using the classification selector next to the type dropdown. For client- and project-scoped uploads, classification is locked to Client (confidential). It can also be changed afterwards using the ✎ edit button.

Library-to-conversation bridge

Technical library documents can be attached directly to the active conversation:

From Library
Click "Add to conversation" on any document row in the library browser. The document is attached to the current session scope. If no conversation is active, a new one is created.
From Chat
Use "Browse library" in the chat sidebar to open the library with a filter. Selected documents are attached to the current conversation.

Knowledge graph & concept linking

When a document is uploaded, LanceLLMot does two things beyond basic vector search:

Citation links
Standard identifiers (IEC 61508-1, ISO 13849, etc.) are extracted by regex and linked as hub nodes. Documents referencing the same standard are connected.
Concept links
The AI extracts abstract safety concepts from each chunk — "diagnostic coverage", "safe state", "proof test interval", etc. Chunks addressing the same concept across different documents are linked regardless of explicit citation numbers.

This means asking "Does this THEOP satisfy IEC 61508 SIL 2 requirements for diagnostic coverage?" can traverse from your THEOP chunks to IEC 61508 chunks that address the same concept, even if the THEOP never cites the standard by number.

Concept extraction runs in the background after upload. A 30B+ model gives the best extraction quality — use the Analysis model selector in the header.

Supported file formats

PDF, DOCX, XLSX, XLS, CSV, TXT, MD, and IEC 61131-3 PLC source files (.st, .scl, .lad, .fbd, .il, .sfc). Maximum file size: 20 MB.
